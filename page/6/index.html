linux账号管理与权限

发表于 2025-05-14 更新于 2026-05-13 分类于 computer

1.linux用户和组

1.1.账号分类

超级管理员：root，UID=0，最高权限

系统用户：UID 1~999，系统进程、服务专用，不能登录

普通用户：UID 1000 起，日常登录使用

1.2.核心配置文件

文件 作用
/etc/passwd 用户账号基本信息（所有人可读）
/etc/shadow 密码、有效期（仅 root 可读）
/etc/group 用户组配置
/etc/gshadow 组密码、组管理员

2./etc/passwd 字段详解

每行一个用户，冒号分隔 7 段

格式：

  1. 账号名
  2. 密码占位符 x（真正密码在 shadow）
  3. UID 用户 ID
  4. GID 初始组 ID
  5. 用户注释信息
  6. 家目录：/home/用户名
  7. 登录 Shell：/bin/bash 可登录，/sbin/nologin 不可登录

3./etc/shadow 字段详解

存放加密密码、过期规则，普通用户无权限看

  1. 账号名
  2. 加密密码（!! 表示无密码、不能登录）
  3. 密码最后修改日期
  4. 密码最短使用天数
  5. 密码最长有效期
  6. 过期前警告天数
  7. 过期后宽限天数
  8. 账号失效日期
  9. 保留字段

7.Linux文件权限底层详解

7.1.权限身份

文件对三种身份设权限：

所有者 u：文件拥有者

所属组 g：同组用户

其他人 o：非所有者、非同组

7.2.三种权限

权限 字符 数字 对文件作用 对目录作用
读 r 4 查看内容 列出目录列表
写 w 2 修改内容 增删改名文件
执行 x 1 程序可运行 可进入目录、切换 cd

目录必须要有 x 权限才能 cd 进入

3. 权限数字计算

r=4 w=2 x=1

例：

  • rwx = 4+2+1 = 7
  • r-x = 4+1 = 5
  • rw- = 4+2 = 6

常见权限：

  • 755：u=rwx g=rx o=rx
  • 644：u=rw g=r o=r

9.特殊权限 SUID/SGID/SBIT

9.1.SUID 4xxx

  • 作用：普通用户执行该文件时，临时拥有文件所有者权限
  • 仅对二进制可执行文件有效
  • 数字：4755

9.2.SGID 2xxx

  • 文件：执行时拥有所属组权限
  • 目录：目录内新建文件自动继承目录所属组
  • 数字：2755

9.3.SBIT 粘滞位 1xxx

  • 只对目录生效，典型：/tmp
  • 作用：任何人可在目录建文件，只能删自己的文件，不能删别人的
  • 数字：1777

10.默认权限 umask

10.1.作用

控制新建文件 / 目录的默认权限

查看：

规则：

  • 目录默认最大 777 减 umask
  • 文件默认最大 666 减 umask

示例：umask=022

  • 新建目录：777-022 = 755
  • 新建文件：666-022 = 644
