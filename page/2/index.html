linux之工作调度

发表于 2025-06-27 更新于 2026-05-28 分类于 computer

Linux 工作调度核心分两类：一次性任务（at）、周期性任务（cron），还有系统级临时任务 anacron，是运维自动化、定时备份、日志切割的核心工具。

一、一次性任务：at 命令（atd 服务）

1. 核心概念

  • 作用：指定未来某一时间点执行一次命令 / 脚本，执行后自动删除任务。
  • 依赖：atd 后台服务必须运行，否则任务无法触发。
  • 典型场景：临时重启、延迟执行脚本、非重复定时操作。

2. 服务管理（CentOS/RHEL）

3. 基本语法与时间格式

常用时间格式（灵活）

  • 绝对时间：HH:MM（如 14:30）、YYYY-MM-DD HH:MM
  • 相对时间：now + 5 minutes、now + 2 hours、now + 1 day
  • 特殊时间：midnight（午夜）、noon（中午）、teatime（16:00）

6. 权限控制（允许 / 拒绝用户）

  • /etc/at.allow：白名单，仅文件内用户可使用 at（优先级高）
  • /etc/at.deny：黑名单，禁止文件内用户使用 at（默认所有用户可用）
  • 规则：有 allow 则仅 allow 用户可用；无 allow 则看 deny；都无则全部可用

二、周期性任务：cron 命令（crond 服务）

1. 核心概念

  • 作用：周期性重复执行命令 / 脚本（分钟、小时、天、周、月），运维最常用。
  • 依赖：crond 服务（系统默认开机自启）。
  • 典型场景：每日备份、日志切割、定时清理缓存、监控告警。

2. 服务管理

3. 核心配置文件

1）用户级任务（个人定时）

  • 路径：/var/spool/cron/用户名（自动生成）
  • 命令：crontab -e 编辑，无需手动改文件

2）系统级任务（全局定时，root）

  • /etc/crontab：主配置，可指定执行用户
  • /etc/cron.d/：自定义任务文件（如系统更新、日志）
  • /etc/cron.hourly/：每小时执行脚本
  • /etc/cron.daily/：每日执行（日志切割默认放这）
  • /etc/cron.weekly/：每周执行
  • /etc/cron.monthly/：每月执行

4. crontab 语法（重点：5 个时间字段）

特殊符号（核心）

  • *：任意（每），如 * * * * * 每分钟执行
  • ,：多个值，如 0,30 * * * * 每小时 0 分、30 分执行
  • -：范围，如 9-18 * * * * 9 点到 18 点每小时执行
  • */n：间隔，如 */10 * * * * 每 10 分钟执行

三、anacron：开机补执行错过的 cron 任务

1. 作用

  • cron 依赖系统持续运行，关机则错过任务；
  • anacron 是开机补执行工具，专门处理关机期间错过的 cron 任务（适合笔记本、桌面服务器）。

四、工作调度常见问题与排错

1. 任务不执行？先查 4 点

  1. 服务是否运行：systemctl status crond/atd
  2. 路径问题：cron 无环境变量，命令用绝对路径（如 /usr/bin/rm）
  3. 权限问题：脚本有执行权限（chmod +x 脚本）
  4. 日志排错：cron 日志默认 /var/log/cron，at 日志 /var/log/at

特性 at（一次性） cron（周期性）
