一、程序、进程、线程区别

  1. 程序 program

    放在硬盘里的静态文件，没运行就是程序。

    比如 /bin/ls、/usr/bin/bash

  2. 进程 process

    程序被运行起来，在内存中工作，就是进程。

    系统每运行一个程序，就会产生一个 PID 进程号。

  3. 线程 thread

    进程里的子任务，一个进程可以多个线程同时工作。

总结：程序 = 文件，进程 = 运行中的程序

二、进程 PID 与 PPID

三、查看进程命令

1. ps 静态查看（快照）

常用组合

字段含义（重点）

  • USER：进程所属用户
  • PID：进程号
  • %CPU：CPU 占用
  • %MEM：内存占用
  • VSZ：虚拟内存
  • RSS：实际物理内存
  • TTY：终端
  • STAT：进程状态
  • START：启动时间
  • COMMAND：命令

进程状态 STAT

  • R：正在运行
  • S：休眠、等待唤醒
  • D：不可被唤醒（磁盘 IO）
  • T：暂停、后台停止
  • Z：僵尸进程（程序结束，父进程没收尸）
  • +：前台进程

2. top 动态实时查看（重点）

每 3 秒刷新一次，相当于任务管理器

常用按键

  • 1：显示所有 CPU 核心
  • M：按内存排序
  • P：按 CPU 排序
  • q：退出

3. pstree 进程树

4. pgrep 只查进程名

四、前台进程 & 后台进程

1. 前台 foreground

直接在终端运行，占据屏幕，不能输其他命令

比如 ping baidu.com

2. 后台 background

程序运行在后台，不占用终端，可以继续操作

3. 前后台切换快捷键

五、进程信号（kill 命令）

作用：控制进程、关闭、重启、强制杀死

最常用信号

信号 数字 含义
SIGHUP 1 重新加载配置、重启进程
SIGINT 2 键盘 Ctrl+c 中断
SIGQUIT 3 退出
SIGKILL 9 强制杀死，必死无疑
SIGTERM 15 正常终止（默认）
SIGSTOP 19 暂停进程

Linux 进程不是一起抢 CPU，有优先级

数字越小，优先级越高

九、僵尸进程 与 孤儿进程

  1. 僵尸进程 Z

    子进程结束，父进程没回收，一直占 PID

    危害：PID 号被占满，系统无法新建进程

  2. 孤儿进程

    父进程意外挂了，子进程没人管
