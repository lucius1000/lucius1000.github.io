3.Etcd使用

Etcd是一个分布式、高可用、强一致性的键值存储系统，由CoreOS公司开发（后被CNCF托管）。

核心定位：

  • 分布式系统的配置中心、服务注册发现中心、分布式锁中心
  • Kubernetes 默认的唯一持久化存储（存储集群所有元数据）

核心特性：

​ 强一致性：基于 Raft 共识算法，保证多节点数据完全一致

​ 高可用：多节点集群，部分节点故障不影响服务

​ 键值存储：简单的 key=value 数据模型，支持版本控制、监听

​ 安全可靠：支持 TLS 加密、权限认证

​ 轻量：资源占用低，部署简单

具有以下应用场景：

​ 服务注册与发现：微服务注册地址，客户端发现服务

​ 配置中心：统一管理分布式服务配置，实时推送更新

​ 分布式锁：解决分布式系统并发竞争问题

​ 集群选主：通过 Raft 实现主节点选举

​ Kubernetes 存储：存储集群节点、Pod、Service 等所有核心数据

3.1.概念简述

1.Raft 算法

etcd 使用 Raft 实现分布式一致性：

  • Leader：主节点，处理所有写请求
  • Follower：从节点，同步数据，被动响应
  • Candidate：候选节点，Leader 故障时参与选举

2.重要术语

  • Revision：数据版本号，每次修改自增，支持历史版本查询
  • Lease：租约，给 key 设置过期时间，类似 Redis 的 TTL
  • Watch：监听 key 变化，实时获取变更通知
  • Cluster：etcd 集群，推荐 3/5/7 奇数节点，容错性最强

3.2.部署

我这边就找一台linux服务器使用docker部署一个etcd服务完成项目实践。

注意：etcd 生产环境必须集群部署。单节点容易故障，出现宕机、断电、进程崩溃、磁盘损坏，所有读写请求直接失败，依赖 etcd 的服务（如 K8s、微服务）全部异常。单节点无法应对硬件故障、系统升级、运维停机，完全不满足生产持续运行要求。
