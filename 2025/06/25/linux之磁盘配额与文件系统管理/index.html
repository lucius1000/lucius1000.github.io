linux之磁盘配额与文件系统管理

发表于 2025-06-25 更新于 2026-05-25 分类于 computer 阅读次数：

一、磁盘配额（Quota）

1. 作用与适用场景

磁盘配额（Quota）：限制用户 / 用户组在分区内可使用的磁盘容量、文件数量，防止单人占用全部磁盘资源。

  • 适用：多人共用服务器、FTP、共享目录
  • 限制维度：
    1. 容量 (block)：占用磁盘大小
    2. 文件数 (inode)：创建文件 / 目录总个数
  • 限制级别：
    • 软限制 (soft)：警告阈值，可临时超出，有宽限期
    • 硬限制 (hard)：绝对上限，到达后无法再写入 / 新建文件

2. linux文件系统

​ 文件系统是操作系统管理磁盘数据的规则与结构。

2.1. 文件系统核心特性

1.一切皆文件：硬件、目录、进程、套接字、设备都以文件形式访问

2.单一根目录：整个系统只有一个根 /，所有分区、磁盘都挂载到根目录下，无 Windows C/D 盘符概念

3.目录树结构：树形层级目录，从 / 向下延伸

2.2. linux目录结构

目录 作用说明
/ 根目录，所有目录起点
/bin 普通用户可执行基础命令（ls、cp、cd 等）
/sbin 系统管理员命令（开机、修复、运维命令，多为 root 使用）
/boot 开机引导文件、内核、grub 引导程序
/dev 设备文件（硬盘、分区、光驱、终端等硬件抽象）
/etc 系统、服务、软件配置文件目录
/home 普通用户家目录，/home/用户名
/lib /lib64 系统运行依赖的库文件、模块
/media 自动挂载点：U 盘、光盘、移动硬盘
/mnt 临时手动挂载目录（挂载分区、共享目录常用）
/opt 第三方额外软件安装目录
/proc 伪文件系统，实时存放内核、进程运行信息（不占磁盘空间）
/root root 管理员家目录
/run 运行时临时数据，进程 PID、套接字，重启清空
/tmp 临时文件目录，所有用户可读写，定期清理
/usr 系统软件、文档、帮助、源码、应用程序（最大目录之一）
/var 动态变化数据：日志、缓存、数据库、邮件

2.3. 文件系统核心组成（Ext/XFS 通用底层结构）

一块格式化后的分区，被文件系统划分为三大核心部分：

1.超级块（Super Block）整个文件系统的总描述信息。

记录：文件系统类型、总容量、块大小、空闲块数量、挂载状态、日志信息、UUID

超级块损坏 → 整个分区无法挂载、数据丢失

为防损坏，系统会多处备份超级块

2.Inode（索引节点）记录文件元数据（属性信息），不存储文件内容。每一个文件 / 目录对应唯一 inode 编号。

Inode 包含信息

  • 文件类型、权限、UID、GID
  • 文件大小、硬链接数
  • 三个时间戳：访问时间 (atime)、修改时间 (mtime)、状态改动时间 (ctime)
  • 数据块指针：指向真实存放内容的磁盘块

Inode特点:

​ 同分区内 inode 号唯一，跨分区可重复

​ 新建文件就占用一个 inode，磁盘满分两种情况：1.空间不足：block 耗尽。2.文件过多：inode 耗尽（小文件大量场景常见）

​ 磁盘配额中 inodes 限制，就是限制文件 / 目录总数

查看Inode

3.Block（数据块）

  • 真正存放文件内容的磁盘最小单位
  • 格式化时指定块大小（常见 1K/2K/4K）
  • 一个大文件会占用多个 block；极小文件也至少占

二、进阶文件系统管理

1. 文件系统日志：日志式文件系统

传统 ext2 无日志，异常断电易损坏文件系统。

  • ext3/ext4/xfs 均为日志文件系统
  • 作用：记录文件读写操作日志，宕机后快速修复，减少数据丢失
  • 日志模式（ext3）：
    1. journal：完整日志，性能低、安全性高
    2. ordered：默认，仅记录元数据，均衡性能与安全
    3. writeback：只回写，性能高、安全性弱

2. 文件系统检测与修复

断电、异常关机导致文件系统损坏，使用 fsck 修复。

⚠️ 重要：修复时分区必须卸载，严禁在挂载状态执行！

  • 开机自检：系统异常关机，开机自动执行 fsck
  • /etc/fstab 最后一列：设置开机自检优先级

3. 调整文件系统大小（resize2fs /xfs_growfs）

场景：分区扩容后，同步放大文件系统

1）ext2/ext3/ext4：resize2fs
